# Complaint Submission Confirmation Fix

## Problem
After submitting a complaint through the AI Assistant, the confirmation message was showing:
- Generic department name from AI service (not the real assigned department)
- Missing department contact details (phone, email)
- Incorrect complaint ID format
- No real SLA hours from backend

## Solution Implemented

### Updated Flutter Code (`ai_chat_screen.dart`)

Modified the `_handleSubmitComplaint()` method to extract real data from backend response:

```dart
// Extract real data from backend response
final complaintResponse = result['complaint'] as Map<String, dynamic>?;
final complaintId = complaintResponse?['complaint_number'] ?? 'Unknown';

// Get real department info from backend
final departmentData = complaintResponse?['assigned_department'] as Map<String, dynamic>?;
final assignedDepartment = departmentData?['name'] ?? 'Municipal Corporation';
final departmentPhone = departmentData?['phone'] ?? '';
final departmentEmail = departmentData?['email'] ?? '';
final slaHours = departmentData?['sla_hours'] ?? 48;
```

The backend API (`/api/complaints/`) returns:

## Benefits

✅ Shows real complaint ID generated by backend  
✅ Displays actual assigned department name  
✅ Includes department contact information (phone & email)  
✅ Shows real SLA hours from department configuration  
✅ Displays priority from backend response  
✅ User can immediately contact the department if needed  

## How Department Assignment Works

1. **Backend Logic** (`complaints/models.py`):
   - When complaint is created, `save()` method is called
   - `get_nearest_department()` finds closest department based on:
     - Complaint category/type
     - Geographic location (latitude/longitude)
     - Department availability
   - Department is automatically assigned

2. **Flutter Receives**:
   - Complete complaint object with nested department data
   - All department contact details
   - SLA hours for estimated resolution time
